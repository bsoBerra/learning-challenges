# Learning Challenges

Технічна карта проєкту для швидкої орієнтації в логіці та назвах компонентів.

## Загальна структура

Проєкт є статичним frontend-застосунком без збірки:

- `index.html` - DOM-структура всіх екранів.
- `styles.css` - всі стилі, responsive layout, вигляд кнопок, годинника і футера.
- `index.js` - координатор між списком випробувань і окремими компонентами випробувань.
  Також відповідає за синхронізацію стану застосунку з URL hash.
- `challenges/clock/clockChallenge.js` - вся логіка компонента випробування годинник.
- `challenges/multiplication/multiplicationChallenge.js` - логіка компонента випробування множення.

Для локального перегляду достатньо відкрити `index.html` у браузері.

## Компонентна структура

Проєкт варто описувати як набір великих екранних компонентів:

- **Компонент списку випробувань** - стартовий екран `#startScreen`, клас `.start-screen`.
  Це головний екран, на якому плитками показаний перелік усіх випробувань-челенджів.
- **Компонент випробування годинник**.
  Містить усе, що стосується випробування годинника. Складається з основного екрана годинника `#clockChallengeScreen` і екрана результату `#clockResultScreen`.
  Основний екран має header, кнопку `На головну`, циферблат, панель керування, рахунок, введення відповіді, перевірку і завершення.
  Екран результату є внутрішнім екраном цього компонента, а не окремим великим компонентом.
- **Компонент випробування множення** - екран `#multiplicationScreen`, клас `.multiplication-screen`.
  Містить усе, що стосується випробування множення. Поки реалізований тільки header і кнопка `На головну`.
- **Компонент футера контактів** - `.site-footer`.
  Загальний футер, має бути видимим на всіх екранах.

Поведінка така: користувач відкриває компонент списку випробувань, натискає плитку конкретного випробування і переходить на компонент цього випробування.

Код кожного випробування має жити максимально окремо:

- компонент списку випробувань і маршрутизація між компонентами - `index.js`;
- компонент годинника - `challenges/clock/clockChallenge.js`;
- компонент множення - `challenges/multiplication/multiplicationChallenge.js`.

Щоб видалити випробування, потрібно прибрати його екран/плитку з `index.html`, його файл у `challenges/...`, підключення script у `index.html` і виклик у `index.js`.

## Плитки випробувань

Плитки живуть тільки всередині компонента списку випробувань:

- **Плитка "Випробування годинник"** - кнопка `#clockChallengeButton`.
  Відкриває компонент випробування годинник.
- **Плитка "Випробування множення"** - кнопка `#multiplicationButton`.
  Відкриває компонент випробування множення.

Активність екранів керується класом `.app-hidden`: якщо клас є, екран прихований.

## Спільні UI-терміни

- **Header випробування** - `.challenge-screen-header`.
  Верхній рядок екрана випробування: заголовок зліва, кнопка `На головну` справа.
- **Плитки випробувань** - кнопки всередині `.challenge-grid`.
- **Основна кнопка** - `.show-button`.
- **Вторинна кнопка** - `.secondary-button`, зараз використовується для `На головну`.
- **Панель керування годинником** - `.controls`.
  Правий блок на екрані годинника з рахунком, вводом часу та кнопками дій.
- **Блок рахунку** - `.score`.
  Показує `Генерацій` і `Рахунок`.
- **Поле відповіді часу** - `.answer-time`.
  Чотири цифрові контролери для введення години і хвилин.
- **Цифровий результат часу** - `#digitalTime`.
  Показує правильний час після натискання `Перевірити`.

## Логіка екранів у JS

Ключові DOM-змінні:

- `challengeListScreen` - компонент списку випробувань.
- `clockChallenge` - контролер компонента випробування годинник.
- `multiplicationChallenge` - контролер компонента випробування множення.

Основні переходи:

- Клік по `#clockChallengeButton`:
  викликає `showClockChallenge()`.
- Клік по `#multiplicationButton`:
  викликає `showMultiplicationChallenge()`.
- Клік по кнопці `На головну` всередині випробування:
  викликає `showChallengeListScreen()`, скидає стан випробування і повертає на головний екран.

Кожен компонент випробування повертає мінімальний публічний API:

- `show()` - показати компонент;
- `hide()` - приховати компонент.
- `reset()` - скинути стан компонента до стартового.

Компоненти, які мають власний стан, можуть додавати до API:

- `getState()` - віддати стан для збереження в URL;
- `setState(state)` - відновити стан із URL.

Внутрішні екрани компонента годинника:

- `#clockChallengeScreen` - основний екран годинника;
- `#clockResultScreen` - екран результату годинника.

## Стан у URL

Застосунок зберігає актуальний стан у `location.hash` у форматі:

- `#state=<base64url>`

Кодування робить `index.js`:

- `encodeUrlState(state)` - JSON -> UTF-8 -> base64url;
- `decodeUrlState(value)` - base64url -> UTF-8 -> JSON;
- `updateUrlState()` - оновлює hash через `history.replaceState`;
- `readUrlState()` - читає hash при завантаженні сторінки.

У URL зберігається:

- активний екран: `list`, `clock` або `multiplication`;
- стан компонента годинника: загаданий час, відповідь користувача, рахунок, кількість генерацій, режим перевірки і внутрішній екран `challenge/result`.

Після оновлення сторінки `index.js` читає URL, викликає `clockChallenge.setState(...)` і відкриває потрібний екран.

## Логіка випробування годинника

Стан зберігається в об'єкті `state`:

- `hour` - поточна загадана година, `0..11`.
- `minute` - поточна загадана хвилина, `0..59`.
- `answerHour` - година, яку ввів користувач.
- `answerMinute` - хвилина, яку ввів користувач.
- `isChecked` - чи вже перевірили поточну відповідь.
- `score` - рахунок.
- `generationCount` - номер генерації часу.

Основні функції:

- `setRandomTime()` - генерує новий випадковий час.
- `renderHands()` - повертає стрілки годинника під `state.hour` і `state.minute`.
- `renderAnswerTime()` - оновлює цифри у полі відповіді.
- `changeAnswerDigit(step, direction)` - змінює одну цифру відповіді.
- `renderAnswerResult(isCorrect)` - показує знак правильно/неправильно.
- `clearAnswerResult()` - очищає знак перевірки.
- `changeScore(delta)` - змінює рахунок і може завершити випробування.
- `updateClaimPrizeButton()` - активує кнопку завершення, коли рахунок більший за 30.
- `setCheckedMode(isChecked)` - блокує/розблоковує введення після перевірки.

Правила рахунку:

- `Перевірити`: правильна відповідь `+2`, неправильна `-1`.
- `Наступний`: якщо попередній час не був перевірений, `-1`.
- Рахунок не падає нижче `0`.
- Якщо `score > 60`, відкривається внутрішній екран результату компонента годинника.
- Якщо `generationCount > 50`, відкривається внутрішній екран результату компонента годинника.

## Layout-нотатки

Сторінка зроблена так, щоб спочатку все намагалось вміститись у viewport, а скрол з'являвся тільки коли місця вже не вистачає:

- `body` використовує `100dvh`.
- Відступи винесені в `--page-padding` і `--page-gap`.
- Розмір годинника обмежений і шириною, і висотою viewport.
- Футер є окремою нижньою строкою layout і не належить до конкретного екрана.
